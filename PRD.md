## 1. 项目概述

### 1.1 项目背景

当前金融管理平台积累了大量的交易数据、客户数据、风险数据和运营数据，但缺乏统一的数据展示与分析能力。业务方在以下方面面临痛点：

- 报表制作依赖人工 Excel，效率低、易出错、无版本追溯
- 管理层无法一站式查看核心经营指标，决策依赖多系统切换
- 业务人员缺乏自助式 BI 分析工具，每次数据需求都需要研发介入

### 1.2 项目目标

构建一个全面的金融数据报表与可视化平台，通过 **报表中心**、**数据大屏**、**智能 BI** 三大核心模块，帮助金融管理平台更好地理解和利用数据，提升客户的数据分析和决策能力。

### 1.3 产品定位

面向金融管理平台的中后台数据产品，服务于管理层、运营人员、分析师等多角色用户，提供从「数据查看 → 报表制作 → 可视化展示 → 智能分析」的全链路数据能力。

## 3. 功能需求

### 3.1 报表中心 (Report Center)

报表中心是平台的基础模块，提供标准报表的管理、生成、订阅与导出能力。

#### 3.1.1 报表目录

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 报表分类树 | 按业务域（财务、风控、运营、客户等）组织报表，支持自定义分类 | P0 |
| 报表搜索 | 按报表名称、标签、创建人、时间范围搜索 | P0 |
| 收藏与最近使用 | 支持收藏常用报表，展示最近浏览/使用的报表列表 | P1 |
| 报表权限 | 按角色/用户控制报表的可见、编辑、导出权限 | P0 |

#### 3.1.2 报表模板

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 预置模板库 | 提供金融行业常用模板：资产负债表、利润表、现金流量表、逾期分析表、客户画像表等 20+ 模板 | P0 |
| 自定义模板 | 用户可基于模板修改并保存为个人/团队模板 | P1 |
| 模板版本管理 | 支持模板的版本历史与回滚 | P2 |

#### 3.1.3 报表生成

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 参数化生成 | 选择模板后，配置时间范围、数据维度、筛选条件等参数，一键生成报表 | P0 |
| 定时生成 | 支持日报、周报、月报的定时自动生成 | P1 |
| 增量更新 | 支持大数据量报表的增量刷新，而非全量重跑 | P2 |

#### 3.1.4 报表查看与交互

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 表格展示 | 支持冻结表头、列排序、列筛选、列宽拖拽、分页/虚拟滚动 | P0 |
| 图表联动 | 表格与图表（柱状图、折线图、饼图）联动展示 | P0 |
| 数据下钻 | 从汇总数据逐层下钻到明细数据（如：全国 → 省 → 市） | P1 |
| 条件格式 | 支持数据高亮（如负值标红、超出阈值标黄） | P1 |
| 数据对比 | 支持同比、环比、自定义周期对比 | P1 |

#### 3.1.5 导出与分发

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 多格式导出 | 支持 PDF、Excel、CSV 格式导出 | P0 |
| 水印与加密 | 导出文件自动添加水印（用户名+时间），敏感报表支持密码保护 | P1 |
| 邮件/消息订阅 | 报表生成后自动推送至邮箱或站内消息 | P1 |
| 打印适配 | PDF 导出适配 A3/A4 纸张打印 | P2 |

| 组件配置 | 每个组件支持数据源绑定、样式配置、交互配置 | P0 |

智能 BI 模块面向分析师和业务人员，提供自助式数据探索与分析能力。

#### 3.3.1 数据源管理

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 数据源接入 | 支持 MySQL、PostgreSQL、ClickHouse、Elasticsearch、API 接口等数据源 | P0 |
| 数据源管理 | 数据源的增删改查、连接测试、状态监控 | P0 |
| 数据权限 | 按用户/角色控制数据源和表的访问权限 | P0 |

#### 3.3.2 数据集

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| SQL 数据集 | 通过 SQL 查询创建数据集，支持参数化 | P0 |
| 拖拽数据集 | 可视化拖拽方式选择表和字段，自动生成 SQL | P1 |
| 多表关联 | 支持 JOIN 多表关联建模 | P1 |
| 数据预览 | 数据集预览（前 100 行），字段类型自动识别 | P0 |
| 计算字段 | 支持自定义计算字段（聚合函数、数学运算、条件判断等） | P1 |
| 缓存策略 | 数据集的缓存配置（缓存时长、手动刷新） | P2 |

#### 3.3.3 自助分析

| 功能点 | 描述 | 优先级 |

| 图表可视化 | 选择字段后推荐最佳图表类型，支持手动切换 | P0 |
| 筛选与过滤 | 支持维度筛选、度量筛选、日期范围筛选、Top N 筛选 | P0 |

#### 3.3.5 AI 辅助分析（远期）

## 4. 非功能性需求

### 4.1 性能要求

| 指标 | 要求 |
|------|------|
| 页面首屏加载 | ≤ 2 秒（P95） |
| 报表生成 | 10 万行以内 ≤ 5 秒；100 万行 ≤ 30 秒 |
| 大屏渲染 | 组件加载完成 ≤ 3 秒，图表渲染 FPS ≥ 30 |
| 数据查询 | 简单查询 ≤ 1 秒，复杂聚合 ≤ 10 秒 |
| 并发用户 | 支持 500+ 并发在线用户 |

### 4.2 安全要求

- 全站 HTTPS，敏感数据传输加密
- 基于 RBAC 的权限控制（页面级、数据源级、行级）
- 操作日志审计（谁在什么时间做了什么操作）
- 导出文件自动添加水印，支持下载溯源
- 防 SQL 注入、XSS、CSRF 攻击

### 4.3 可用性要求

- 系统可用性 ≥ 99.9%（全年停机时间 < 8.76 小时）
- 支持主备容灾，故障自动切换

### 4.4 兼容性要求

- 浏览器：Chrome 90+、Edge 90+、Firefox 90+、Safari 15+
- 屏幕分辨率：最低支持 1366×768，大屏推荐 1920×1080 及以上

## 10. 风险与假设

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 数据源兼容性复杂 | 接入多种数据源时适配工作量大 | 先支持 MySQL/PG，逐步扩展 |
| 大数据量性能瓶颈 | 报表生成慢、大屏卡顿 | 引入 ClickHouse/ES 做查询加速，数据预聚合 |
| AI 功能可行性不确定 | 自然语言转 SQL 准确率低 | 作为远期规划，先做好规则引擎兜底 |
| 第三方图表库限制 | ECharts 大屏场景性能不足 | 预研 AntV/G2 或 WebGL 方案作为兜底 |
